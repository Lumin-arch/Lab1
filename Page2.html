Shell & Bash Basics

The shell is the primary interface between you and the Linux kernel. Mastering it is the single most important skill in your journey — whether you're doing system administration, penetration testing, or software development.

bash shell filesystem permissions cli

What is a Shell?

A shell is a command-line interpreter that exposes the operating system's services to the user. When you open a terminal emulator, you're launching a shell process that reads your input, evaluates it, and returns output. Think of it as a REPL (Read-Eval-Print Loop) for your entire computer.

The most common shell on modern Linux systems is bash — the Bourne Again SHell. It extends the original Bourne shell with features like command history, tab-completion, and arithmetic expansion. Other popular alternatives include zsh, fish, and sh.

Note

On most Debian/Ubuntu systems, the default interactive shell for users is bash, while /bin/sh points to dash — a lighter POSIX-compliant shell used for system scripts.

Check your current shell

bash 
# Print your current shell
echo $SHELL
/bin/bash

# List all available shells on the system
cat /etc/shells
# /etc/shells: valid login shells
/bin/sh
/bin/bash
/usr/bin/bash
/bin/zsh

Bash Commands

Every bash command follows a consistent structure. Understanding this anatomy makes you faster at reading man pages and deciphering commands you've never seen before.

Command Anatomy

↑ command ↑ flags/options ↑ argument/path

Piping & Redirection

The Unix philosophy of "small tools that do one thing well" is made powerful by pipes (|) and redirection operators (>, >>, <). They allow you to chain commands into powerful one-liners. 

# Pipe: send stdout of one command into stdin of another
cat /etc/passwd | grep -i "bash" | cut -d':' -f1
root
htbuser
kali

# Redirect stdout to a file (overwrites)
ls -la > listing.txt

# Redirect stdout and append to file
echo "new entry" >> listing.txt

# Redirect stderr to /dev/null (suppress errors)
find / -name "*.conf" 2>/dev/null

Warning

Using > redirection on an existing file will overwrite it silently. Always double-check the target filename before running. Use >> if you want to append instead.

File System Navigation

The Linux file system is a single hierarchical tree rooted at /. Unlike Windows, there are no drive letters — everything hangs off this single root. Understanding which directories serve which purpose is essential for both administration and offensive security work.

Filesystem Hierarchy Standard (FHS)

/
├── bin → essential user binaries
├── boot → bootloader files, kernel
├── dev → device files
├── etc → system-wide config files ⭐
├── home → user home directories
├── lib → shared libraries
├── opt → optional/third-party software
├── proc → virtual FS: running processes ⭐
├── root → root user home dir
├── tmp → temporary files (world-writable!) ⚠️
├── usr → user programs & data
└── var → variable data: logs, mail ⭐
# Navigate directories like a pro
cd /var/log          # absolute path
cd ..              # one level up
cd ~               # home directory
cd -               # previous directory (toggle!)

# Enhanced directory listing
ls -lah /etc/
total 1.3M
drwxr-xr-x  1 root root  4.0K May 12 08:21 .
drwxr-xr-x  1 root root  4.0K May 12 08:21 ..
-rw-r--r--  1 root root  3.0K Jan 10 12:00 passwd
-rw-r-----  1 root shadow 1.1K Jan 10 12:00 shadow
drwxr-xr-x  1 root root  4.0K Jan 10 12:00 ssh

# Find files efficiently
find /home -name "*.txt" -type f 2>/dev/null
find / -perm -4000 -type f 2>/dev/null  # SUID files

Pro Tip

Press Tab for autocomplete and Ctrl+R for reverse history search. These two shortcuts alone will multiply your terminal speed by 10×.

Permissions & Ownership

Linux permissions are one of the most fundamental — and frequently misunderstood — concepts. Every file has an owner, a group, and a set of permission bits for three classes of users: owner, group, and others.

Permission String Breakdown

- rwx r-x r-- root root 4096 /etc/passwd

File Type

— = file
d = directory
l = symlink

Owner (u)

rwx = read, write, execute

Group (g)

r-x = read & execute, no write

Others (o)

r-- = read only

# View permissions
ls -la script.sh
-rw-r--r-- 1 htbuser htbuser 142 May 10 14:22 script.sh

# Make it executable (symbolic notation)
chmod +x script.sh

# Set exact permissions (octal notation)
chmod 755 script.sh   # rwxr-xr-x
chmod 644 file.txt    # rw-r--r--

# Change ownership
sudo chown root:root script.sh
sudo chown htbuser:www-data /var/www/html

# Find SUID/SGID files (critical for privilege escalation!)
find /usr/bin -perm /4000 -ls 2>/dev/null
131073 44 -rwsr-xr-x 1 root root  44784 May  9  2022 /usr/bin/newgrp
131074 68 -rwsr-xr-x 1 root root  67816 Apr  2 14:29 /usr/bin/su
131075 56 -rwsr-xr-x 1 root root  55528 Feb  7  2022 /usr/bin/passwd

Security Alert

SUID binaries run with the owner's privileges (often root), regardless of who executes them. Misconfigured SUID binaries are a classic privilege escalation vector. Always audit them on systems you're testing.
